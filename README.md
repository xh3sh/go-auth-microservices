# 🛡️ Go API Auth System

Микросервисная система аутентификации на **Go (Gin)**. Проект демонстрирует реализацию пяти стратегий входа, асинхронное логирование через **RabbitMQ** и работу с **Redis** в качестве основного хранилища.

---

## 🏗️ Микросервисная архитектура

Система разделена на 4 специализированных сервиса:

1.  **API Gateway (Порт 8080)**: Единая точка входа. Проверяет авторизацию и проксирует запросы.
2.  **Auth Service (Порт 8081)**: Логика входа, JWT, сессии и API-ключи.
3.  **User Service (Порт 8082)**: Управление профилями и пользователями.
4.  **Log Consumer (Порт 8084)**: Обработка событий из RabbitMQ и сохранение в Redis.

---

## 🔑 Способы авторизации

| Метод | Описание | Хранение (TTL) |
| :--- | :--- | :--- |
| **JWT** | Access (1ч) и Refresh (1д) токены в HttpOnly Cookies. | Redis + Cookies |
| **API Keys** | Формат `ID.Secret`. Хешируется перед сохранением. | Redis (24ч) |
| **Sessions** | Классические серверные сессии. | Redis (24ч) |
| **Basic Auth** | Вход по логину/паролю (bcrypt). | Redis (24ч) |
| **OAuth 2.0** | Вход через Google/GitHub (с генерацией JWT). | Redis (24ч) |

---

## ⏱️ Хранение данных (Демо-режим)
Все данные (пользователи, логи, ключи) автоматически удаляются через **24 часа**. Это идеальный режим для демонстрации портфолио.

## ⚙️ Конфигурация (.env)

Система использует гибкую настройку через переменные окружения. Если файл `.env` отсутствует, приложение автоматически использует **стандартные значения** (defaults), прописанные в коде (`internal/config/config.go`).

### Основные переменные и их значения по умолчанию:

| Переменная | Описание | Значение по умолчанию |
| :--- | :--- | :--- |
| **Инфраструктура** | | |
| `REDIS_HOST` | Хост Redis | `localhost` |
| `REDIS_PORT` | Порт Redis | `6379` |
| `RABBITMQ_HOST` | Хост RabbitMQ | `localhost` |
| `RABBITMQ_PORT` | Порт RabbitMQ | `5672` |
| **Порты сервисов** | | |
| `API_GATEWAY_PORT` | Порт шлюза | `8080` |
| `AUTH_SERVICE_PORT` | Порт сервиса авторизации | `8081` |
| `USER_SERVICE_PORT` | Порт сервиса пользователей | `8082` |
| `FRONTEND_PORT` | Порт фронтенда | `8083` |
| `LOG_CONSUMER_PORT` | Порт потребителя логов | `8084` |
| **Безопасность** | | |
| `JWT_SECRET` | Ключ подписи JWT | `your-secret-key...` |
| `JWT_EXPIRATION` | Время жизни Access Token | `3600` (1 час) |
| `JWT_REFRESH_EXPIRATION` | Время жизни Refresh Token | `86400` (1 день) |

> **Примечание:** При запуске в **Docker** переменные `HOST` (например, `REDIS_HOST`) переопределяются именами соответствующих контейнеров в `docker-compose.yml`.

## 🚦 Быстрый старт

### Вариант 1: Полный запуск через Docker (Рекомендуется)
Этот способ запустит все микросервисы в контейнерах. Убедитесь, что у вас создана внешняя сеть, указанная в конфиге:

```bash
# 1. Создаем сеть для взаимодействия контейнеров
docker network create projects-network

# 2. Запускаем инфраструктуру (Redis, RabbitMQ) 
# Примечание: Убедитесь, что они запущены и доступны под именами 'redis' и 'rabbitmq'

# 3. Собираем и запускаем микросервисы
docker-compose up -d --build
```

### Вариант 2: Локальный запуск (Для разработки)
Если вы хотите запускать сервисы через `go run`, сначала запустите только базу и очередь:

```bash
# 1. Запуск инфраструктуры в Docker
docker run -d --name redis -p 6379:6379 redis
docker run -d --name rabbitmq -p 5672:5672 -p 15672:15672 rabbitmq:3-management

# 2. Запуск сервисов (в разных терминалах)
go run cmd/api-gateway/main.go
go run cmd/auth-service/main.go
go run cmd/user-service/main.go
go run cmd/log-consumer/main.go
```

**Доступ к панели:** `http://localhost:8080`
